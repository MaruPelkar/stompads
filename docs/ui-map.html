Landing Page

Client Component
Interactive Elements
  • INPUTURL text input with animated placeholder cycling through sample sites (acmecoffee.com, freshpetfoods.co, novafit.studio, etc.)
  • BUTTON"LAUNCH MY ADS" / "LAUNCHING..." - triggers navigation based on auth state
  • LINKSTOMPADS logo (/) in nav
  • LINK"Dashboard" (if logged in) or "Log in" (if not) in nav
  • LINKFooter: Privacy (/privacy), Terms (/terms)
  • DISPLAY6 floating ad cards (4 images + 1 video + play button) with drift animations
  • DISPLAYHeadline: "RUN ADS. NOT HEADACHES."
  • DISPLAYSubtitle: "Paste your URL. We do the rest. Ads go live in minutes."
  • DISPLAY"Free to try. No credit card. No setup."
State
  • url string - user-entered URL value
  • typingText string - animated placeholder text
  • userFocused boolean - stops animation when user focuses input
  • launching boolean - disables button during navigation
  • isLoggedIn boolean - checked on mount via supabase.auth.getUser()
API Calls
  • AUTH supabase.auth.getUser() on mount (checks login state)
Navigation Logic
On "LAUNCH MY ADS" click:
• URL entered + logged in → /onboard?url={url}
• URL entered + not logged in → /signup?redirect=/onboard?url={url}
• No URL + logged in → /onboard
• No URL + not logged in → /signup

On Enter key: same as button click
Nav link: logged in shows "Dashboard" → /dashboard; not logged in shows "Log in" → /login
/login

Login Page

Client Component
Interactive Elements
  • INPUTEmail (type=email, required, placeholder "you@example.com")
  • INPUTPassword (type=password, required, placeholder "--------")
  • BUTTON"LOG IN" / "LOGGING IN..." (submit)
  • LINK"Sign up" → /signup
  • DISPLAYError message box (conditional)
Layout
Auth layout: centered card, max-w-sm, STOMPADS logo, "Ads on autopilot" tagline.
State
  • email string
  • password string
  • error string | null
  • loading boolean
API Calls
  • AUTH supabase.auth.signInWithPassword({ email, password })
Navigation
On success → /dashboard. Middleware also redirects authenticated users to /dashboard (honors ?redirect param).

/onboard

Onboard Page (Wizard)

Client Component
Components Used
  • UrlForm - URL input + GO button (step: url)
  • AdPreview - Video ad previews (step: preview)
  • BudgetForm - Daily budget input (step: preview)
State
  • step 'url' | 'generating' | 'preview' | 'checkout' - wizard step
  • campaignId string | null - created campaign ID
  • hasProfile boolean - brand profile loaded (progress indicator)
  • ads Ad[] - generated ads
  • error string | null
  • progressIndex number - 0-5 index into progress steps
API Calls
  • POST /api/campaigns/create — { url } → { campaignId }
  • GET /api/campaigns/[id]/status — polled every 3s during generation
  • POST /api/campaigns/[id]/checkout — { dailyBudgetCents } → { checkoutUrl }
Conditional UI - Step by Step

Step: url

  • DISPLAYHeading: "DROP YOUR URL"
  • DISPLAYSubtext: "We handle the rest. Ads go live when you say go."
  • INPUTURL text field (placeholder "yourwebsite.com")
  • BUTTON"GO" (submit) / "ON IT..." (loading)
Auto-starts if ?url query param is present (skips this step, goes directly to generating). Only fires once (autoStarted ref).

Step: generating

  • DISPLAYSpinner (centered)
  • DISPLAY6-step progress indicator with circle markers:
1. "Scanning your website" (scrape)
2. "Locking in your brand" (profile)
3. "Writing ad scripts" (scripts)
4. "Producing video 1" (video1)
5. "Producing video 2" (video2)
6. "Burning in captions" (subtitles)

Progress auto-advances every 12s, plus jumps based on polling: brandProfile received → index 3; 1 ad received → index 4; 2 ads → index 5.
Completed steps: green circle with checkmark. Current step: orange circle, "..." suffix, bold. Future: gray circle.

If hasProfile=true, shows message: "Understood. Creating your ads now."

Step: preview

  • DISPLAYHeading: "ADS READY"
  • DISPLAYAd count: "N video ads -- Instagram Stories & Reels"
  • VIDEOVideo previews in horizontal row (140x248px each, with controls)
  • DISPLAYHeading: "DAILY BUDGET"
  • INPUTBudget number input with $ prefix (default 50, range $10-$10,000)
  • DISPLAY"Enter a number between $10 to $10K"
  • DISPLAY"We take a 10% cut as a publishing platform."
  • BUTTON"CONFIRM & LAUNCH" / "PROCESSING..."
  • DISPLAYValidation error (conditional)

Step: checkout

  • DISPLAYGreen spinner + "Redirecting to payment..."
Browser redirected to Stripe checkout via window.location.href.

/dashboard

Dashboard (Campaign List)

Server Component
Interactive Elements
  • DISPLAYHeading: "YOUR CAMPAIGNS"
  • LINK"+ New campaign" → /onboard
  • LINKEach campaign card (clickable) with hover effect
  • DISPLAYPer card: URL, daily budget "$X/day" or "No budget set"
  • DISPLAYStatus badge (live=green, ready=blue, default=gray)
  • DISPLAY"Resume →" label for draft/payment_pending/ready campaigns
Empty State
  • DISPLAYHeading: "NO CAMPAIGNS YET"
  • DISPLAY"Drop a URL. Get ads. Get traffic."
  • BUTTON"START NOW" → /onboard
Data Fetching (Server)
  • DB supabase.auth.getUser()
  • DB campaigns SELECT where user_id, ordered DESC
Navigation Logic per Status
generating → /dashboard/{id} (show progress)
payment_pending → /dashboard/{id} (resume checkout)
ready → /dashboard/{id} (go live)
preview_ready → /dashboard/{id}
draft → /onboard?url={url} (restart)
live → /dashboard/{id} (view metrics)
paused → /dashboard/{id} (resume)
Status Labels in UI
live → "live"
ready → "ready"
generating / generating_full → "processing"
payment_pending → "awaiting payment"
preview_ready → "ready"
Others → raw status string
/dashboard/[campaignId]

Campaign Detail

Server Component + Client Islands
Components Used
  • GoLiveButton - "GO LIVE" action (client, shown when status=ready)
  • PauseResumeButton - Pause/Resume toggle (client, shown when live or paused)
  • ResumeCheckout - Budget form + checkout redirect (client, shown when payment_pending)
Data Fetching (Server)
  • DB supabase.auth.getUser()
  • DB campaigns SELECT by id + user_id (single)
  • DB ads SELECT by campaign_id, ordered by created_at
  • DB metrics SELECT by ad_ids, ordered by recorded_at DESC (latest per ad)
Always-Visible Elements
  • DISPLAYHeader: product name (or URL as fallback) + URL + status badge
Conditional UI by Campaign Status

Status: live or paused (isRunning)

  • DISPLAYCompact summary bar: "$X/day", "$X.XX spent", "N clicks", "N impressions" (mono font, inline)
  • BUTTON"PAUSE ADS" (ghost, red text) when live / "RESUME ADS" (primary) when paused
Heading uses heading-md (smaller). PauseResumeButton calls POST .../pause with action.

Status: not running + has daily_budget

  • DISPLAY3-column metric cards: Daily Budget "$X/day", Total Spend "$X.XX", Total Clicks "N" (26px display font)

Status: generating

  • DISPLAYCentered spinner + "Still processing."
  • DISPLAY"If stuck, try again."
  • LINK"Retry" → /onboard?url={campaign.url}

Status: draft (failed)

  • DISPLAY"Campaign failed."
  • BUTTON"Try Again" → /onboard?url={campaign.url}

Status: payment_pending (+ has ads)

  • DISPLAY"Ads ready. Set budget to continue." (orange, bold)
  • FORMBudgetForm component (budget input + "CONFIRM & LAUNCH")
ResumeCheckout component: calls POST .../checkout, redirects to Stripe on success.

Status: ready

  • BUTTON"GO LIVE" (22px, 3px letter spacing, full width) / "LAUNCHING..."
  • DISPLAYError box (conditional)
GoLiveButton component: calls POST .../go-live, refreshes page on success.

Ads Section (when ads exist)

  • DISPLAYHeading: "YOUR ADS"
  • DISPLAY2-column grid of LiveAdCard components
Each LiveAdCard shows:
• Video player (280px height, autoplay muted loop) or empty bg
• Type label: "video 9:16" etc. + status badge
• If metrics exist: 2x2 grid of Impressions, Clicks, CTR (orange, percent), CPC (dollar), Spend (dollar, full width)
• If no metrics: "Metrics sync hourly. Check back soon."
/admin/library

Admin - Ad Template Library

Client Component
Interactive Elements
  • DISPLAYHeading: "Ad Template Library"
  • FORM"Add New Template" form (dark bg card):
  • INPUTCategory (text, required, placeholder "e.g. ecommerce, saas, fitness")
  • TEXTAREAPrompt Inspiration (4 rows, required)
  • INPUTNotes (text, optional)
  • INPUTReference Visual (file, accept image/*,video/*, optional)
  • BUTTON"Save Template" / "Saving..." (blue bg)
  • DISPLAYSuccess/error message
  • DISPLAYTemplate list: each card shows category badge, date, prompt snippet (2-line clamp), notes
State
  • items AdLibraryItem[]
  • category string
  • prompt string
  • notes string
  • file File | null
  • loading boolean
  • message string | null
API Calls
  • GET /api/admin/library (on mount, fetches all templates)
  • POST /api/admin/library (FormData: category, prompt, notes, visual file)
Access Control
API routes check if user email matches ADMIN_EMAIL env var. 403 Forbidden otherwise. Page itself is behind /admin route (protected by middleware).
