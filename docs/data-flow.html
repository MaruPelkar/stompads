CAMPAIGN CREATION PIPELINE

URL → Scrape → Brand Profile → Competitor Research → Asset Storage → Video Generation → Save
Step 1
Submit URL
Onboard page calls POST /api/campaigns/create with { url }. Inserts campaign row with status="generating". Returns campaignId immediately.
POST /api/campaigns/create campaigns INSERT
→
Step 2
Trigger Processing
/create fires server-to-server POST to /api/campaigns/[id]/process (non-blocking, forwards auth cookies). Client polls /status every 3s.
POST .../process GET .../status
→
Step 3
Scrape Website
scraper.ts calls Firecrawl with markdown + JSON extraction. Extracts: logo, favicon, product images, brand colors, OG image, title, description.
scraper.ts Firecrawl Langfuse
→
Step 4
Build Brand Profile
brand-profiler.ts sends scrape data to Claude (claude-sonnet-4-6). Returns BrandProfile (category, tone, audience, value props) + AdCopy (headline, primaryText, description).
brand-profiler.ts Anthropic Langfuse
Step 5
Competitor Research
competitor-research.ts searches Meta Ad Library API for similar ads using top 3 value props as keywords. Non-fatal: errors are logged but don't block.
competitor-research.ts Meta Ad Library
→
Step 6
Store Brand Assets
asset-storage.ts downloads external images (logo, OG, products) and re-uploads them to Supabase Storage bucket "ad-creatives". Filters out tracking pixels (<1KB) and huge files (>10MB).
asset-storage.ts Supabase Storage
→
Step 7
Save Profile to DB
Updates campaigns row: brand_profile (JSON), brand_assets (JSON), ad_copy (JSON). Status stays "generating".
campaigns UPDATE
→
Step 8
Generate Video Ads
ad-generator.ts: (1) brand-profiler.ts generates 2 UGC scripts via Claude with different hooks/settings. (2) fal-client.ts generates 9:16 videos via Fal.ai (model from ad_config). (3) Auto-subtitles via Fal.ai. Each ad saved to DB immediately (sequential). Campaign status → "ready".
ad-generator.ts brand-profiler.ts fal-client.ts ad-config.ts Fal.ai Anthropic ads INSERT campaigns UPDATE ad_config SELECT

GO LIVE FLOW

Creates Meta campaign, ad set, and ads via Meta Marketing API. 10% commission deducted, budget in INR.
Step 1
Go Live Button
GoLiveButton.tsx (client component) calls POST /api/campaigns/[id]/go-live. Only available when campaign.status === "ready".
POST .../go-live
→
Step 2
Create Meta Campaign
meta-ads.ts createCampaignWithBudget(). Objective: OUTCOME_TRAFFIC. Status: ACTIVE. Daily budget converted: USD cents * 0.9 (commission) * 85 (USD_TO_INR).
meta-ads.ts Meta Marketing API
→
Step 3
Create Ad Set
createAdSet(): billing=IMPRESSIONS, optimization=LINK_CLICKS, bid=LOWEST_COST_WITHOUT_CAP. Advantage+ targeting (US only). No ad-set-level budget (campaign controls).
meta-ads.ts campaigns UPDATE (meta IDs)
→
Step 4
Create Ads
For each ad: (video) upload video → wait for thumbnail → createVideoAdCreative → createAd. (image) uploadAdImage → createImageAdCreative → createAd. Updates ad rows: meta_ad_id, meta_creative_id, status → "live". Campaign status → "live".
meta-ads.ts Meta Marketing API ads UPDATE campaigns UPDATE

Vercel Cron

Vercel Cron
→
Step 1
Fetch Live Campaigns
GET /api/cron/sync-metrics queries campaigns where status="live" and meta_campaign_id is not null.
GET /api/cron/sync-metrics supabase/server (admin) campaigns SELECT
→
Step 2
Fetch Meta Insights
For each campaign, calls Meta Graph API /{meta_campaign_id}/ads?fields=id,name,insights{impressions,clicks,ctr,cpc,spend}.
Meta Graph API v19.0
→
Step 3
Match & Store
Looks up ads table by meta_ad_id. Inserts new metrics row: impressions, clicks, ctr, cpc, spend, recorded_at.
ads SELECT metrics INSERT
→
Display
Campaign Detail
Campaign detail page fetches latest metrics per ad (ordered by recorded_at DESC, first per ad_id). Shows: impressions, clicks, CTR, CPC, spend.
metrics SELECT

S

CAMPAIGN STATUS STATES

All possible values for campaigns.status and what triggers each transition.
draft
Initial or failed. Set when processing fails at any step.
generating
Created by /create. Processing pipeline running (scrape, profile, video gen).
ready
All ads generated OR payment confirmed. Awaiting "Go Live".
payment_pending
Stripe checkout session created but not yet confirmed.
preview_ready
Legacy/alternate state. Treated same as "ready" in dashboard UI.
live
Meta campaign is ACTIVE. Ads running. Metrics syncing hourly.
paused
Meta campaign is PAUSED. Can be resumed.
generating_full
Legacy state for full generation. Shown as "processing" in UI.

Status Transitions:
draft → generating   (POST /create)
generating → ready   (ad-generator.ts completes)
generating → draft   (any processing step fails)
ready → payment_pending   (POST .../checkout)
payment_pending → ready   (Stripe webhook confirms)
ready → live   (POST .../go-live succeeds)
live → paused   (POST .../pause action=pause)
paused → live   (POST .../pause action=resume)

DB

DATABASE TABLES

Supabase PostgreSQL. Types defined in src/types/database.ts.

profiles

  • id string (PK, matches auth.users.id)
  • email string
  • created_at timestamp

campaigns

  • id string (PK)
  • user_id FK → profiles
  • url string
  • brand_profile JSON | null
  • brand_assets JSON | null
  • ad_copy JSON | null
  • status enum (8 values)
  • daily_budget number | null (cents)
  • meta_campaign_id string | null
  • meta_adset_id string | null
  • stripe_payment_intent_id string | null
  • created_at timestamp

ads

  • id string (PK)
  • campaign_id FK → campaigns
  • type "image" | "video"
  • is_preview boolean
  • fal_request_id string | null
  • asset_url string | null
  • meta_ad_id string | null
  • meta_creative_id string | null
  • aspect_ratio string | null
  • placement string | null
  • status "generating"|"ready"|"live"|"failed"
  • prompt_used string | null
  • created_at timestamp

metrics

  • id string (PK)
  • ad_id FK → ads
  • campaign_id FK → campaigns
  • impressions number
  • clicks number
  • ctr number
  • cpc number
  • spend number
  • recorded_at timestamp

ad_library

  • id string (PK)
  • category string
  • prompt string
  • visual_url string | null
  • notes string | null
  • created_at timestamp

ad_config

  • key string (PK)
  • value string
  • updated_at timestamp

Keys: video_model, video_duration, video_resolution, video_aspect_ratio, subtitle_enabled

E

EXTERNAL SERVICES

All third-party APIs and services the application connects to.

Supabase

  • Auth (email/password signup & login)
  • PostgreSQL (all tables)
  • Storage (ad-creatives, ad-library buckets)
  • 3 client types: browser, server (cookie), admin (service role)

Anthropic (Claude)

  • Model: claude-sonnet-4-6
  • Brand profile + ad copy generation
  • UGC video script writing (2 scripts)
  • Used in: brand-profiler.ts

Firecrawl

  • Website scraping (markdown + JSON extraction)
  • Extracts brand assets, content, metadata
  • Used in: scraper.ts

Fal.ai

  • Image gen: fal-ai/nano-banana-2 (base + edit)
  • Video gen: configurable (default: fal-ai/sora-2/text-to-video)
  • Auto-subtitles: fal-ai/workflow-utilities/auto-subtitle
  • Used in: fal-client.ts

Meta Marketing API

  • Graph API v19.0
  • Campaign, Ad Set, Creative, Ad CRUD
  • Video + image upload
  • Ad Library search (competitor research)
  • Insights fetch (metrics sync)
  • Pause / Resume
  • Used in: meta-ads.ts, competitor-research.ts, sync-metrics route

Stripe

  • Checkout Sessions (one-time payment)
  • Webhook: checkout.session.completed
  • Used in: checkout route, webhook route

Langfuse

  • LLM observability / tracing
  • Traces: scrape, brand-profile, scripts, video gen, subtitles, competitor research
  • Used in: most lib files

Vercel

  • Hosting (Next.js)
  • Cron jobs (hourly metrics sync)
  • maxDuration: 300s (process), 120s (go-live)
